## 4. 분석 코멘트 핵심 패턴 (Analysis Comment Patterns)

레퍼런스 보고서의 분석 코멘트는 단순한 데이터 나열이 아닌, **해석과 스토리텔링**을 통해 데이터를 전달합니다.

### 4.1 구조 패턴
```
[핵심 인사이트], [상세 분석/맥락], [제안 또는 관찰]
```

### 4.2 핵심 원칙

#### 1) 데이터 기반 해석
- **절대 숫자만 나열하지 않음**
- 항상 의미와 맥락 제공
- 예: "전체 57.9%" → "1월과 6월의 특이점으로 인한 것으로 이외 기간 승률은 우수"

#### 2) 구체적인 약점/강점 식별
- 특정 상대 이름 언급 (소심, 나무늘봄숲, 녹대채린, 하이현, 경공)
- 특정 맵 언급 (폴리포이드, 라데온, 데자뷰)
- 특정 종족/매치업 언급 (테란전, 저그전, 동족전)

#### 3) 실행 가능한 개선 제안
- "교육 및 연습이 필요"
- "추가적인 연습이 필요"
- "꾸준한 보완이 필요해 보임"

#### 4) 맥락 제공
- 예외/이상치 설명: "1월과 6월의 특이점으로 인한 것으로", "3월을 제외한"
- 배경 정보: "6월 5일 스폰 시작 이후"
- 비교 분석: "타 종족전에 비해", "스폰보다 12.34%p 높음"

#### 5) 스토리텔링
- 대비/반전/진행 구조 사용
- 예: "대회 및 CK 승률이 대단히 높은 점은 매우 고무적. 하지만 숙달하기는 이르니 조금 더 지켜볼 것"

#### 6) 긍정적 프레이밍
- 잠재력 강조: "고무적", "감사를 전함", "매우 고무적"
- 부족한 부분도 개선 가능한 것으로 표현

#### 7) 유머/친근함
- 캐주얼한 표현: "너무 맞아서", "무드러 맞았던"
- 커뮤니티 친화적 톤
